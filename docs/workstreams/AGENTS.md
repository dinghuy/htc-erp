# workstreams

Parent `AGENTS.md` guidance applies unless this file narrows it for `docs/workstreams`.

## Scope Notes

- `docs/workstreams/` contains only active implementation plans that still direct current execution.
- Every workstream file must be linked from `docs/index.md` with a one-line purpose description.
- Do not keep completed, superseded, or absorbed plans here.
- Archive trigger: when a workstream no longer drives execution because the work shipped, was cancelled, was replaced, or was absorbed into canonical docs, move it to `archive/` and remove or replace its entry in `docs/index.md` in the same change.
- Cross-links inside workstream files may point to canonical docs or sibling workstreams, but routine task startup must still begin from `docs/index.md`.
